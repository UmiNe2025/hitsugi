# 『灯継ぎ -HITSUGI-』プレイヤー魅力向上マスタープラン

**策定日**: 2026-07-21
**対象HEAD**: `74d4fb9`（GitHub Pages公開版）
**位置づけ**: M39後の次期計画。実装、画像生成、commit、push、deployは本依頼に含めない。
**主題**: 「見栄えが良くなったゲーム」を「自分の一族の次を見たくなるゲーム」へ変える。

## 1. エグゼクティブサマリー

『灯継ぎ』は、タイトル画面、郷、出立、ダンジョン背景、主要UIの見た目が大きく改善した。一方、現時点で「プレイヤーにとって魅力が上がった」と断定できる外部証拠はない。新規プレイヤー数、初回帰還率、初回継承率、再訪率、外部8名評価は未測定であり、既存のasset数、test数、closure数は魅力の代理指標にしない。

主経路は、**追加物量を一時凍結し、初回30分と最初の世代交代を一つの因果ループとして磨き、実ユーザーで検証すること**とする。

```text
家祖の危機
  → 星契り
  → 子の誕生・命名
  → 初遠征
  → 手動でもオートでも、育てた一族の戦い方が見える
  → 家族・土地・千年の変化を持ち帰る
  → 約束が家譜へ残る
  → 初代の死
  → 次代が具体的な痕を受け継ぎ、もう一月を選ぶ
```

次の8〜12週間では、40地域や180神の追加制作ではなく、この一本と`発見 → 見返す → 育て方を決める → 次の遠征で確かめる`循環を合格させる。オート戦闘は初遭遇、通常、elite、稀相、主、宿敵、常夜百層を含む**全戦闘で常時利用可能**とし、報酬、発見、称号、育成素材へ手動との差を設けない。初回帰還、初回継承、継承後の自発継続が改善しない場合、全景美術やコンテンツ量産を続けない。

## 2. テーマの解剖

### 2.1 表明されたテーマ

全体的に改善した現在のゲームを、さらにユーザーにとって魅力的にするために何が必要か。

### 2.2 本当のジョブ

プレイヤーが「機能や素材が多い」と評価することではない。短命な一族の一人ひとりを覚え、自分の判断が次代へ残ったと感じ、無理なく続きを選びたくなること。

### 2.3 誰にとっての成功か

- 第一対象: 世代交代、喪失、育成、ローグライトを好む初見プレイヤー。
- 第二対象: 5〜15分の短いブラウザプレイを繰り返すプレイヤー。
- 制作者側: 追加素材の枚数ではなく、露出の高い体験へ労力を集中できること。

### 2.4 暫定成功指標

外部データがないため、以下は仮説検証用の暫定値とする。

- 初見8名中6名が10分以内に星契り→誕生を無介助で完了する。
- 初見8名中6名が30分以内に初回帰還、または安全に中断し次の目的を説明する。
- 5名の一世代テストで初回継承までの中央値45分以内。
- 継承直後、5名中3名以上が促されずに次の一月を始める。
- 初見8名中6名が「この一族の出来事は自分の選択の結果だ」を4/5以上で評価する。
- 経験者40名以上を集められた段階でSean Ellis式の「遊べなくなったらとても残念」40%以上を目安にPMFを判定する。

### 2.5 暗黙の前提

1. 見た目の改善が魅力向上へ直結した。
2. 180神、579敵、810装備、270事件の量が長期継続へつながる。
3. 自動テストとbot勝率が人間の楽しさを代表する。
4. 2〜4時間で遊び切れる。
5. 日参り、日替わり御題、留守番内職が作品固有の継続動機を補強する。

これらはいずれも未証明であり、本計画では事実として扱わない。

## 3. 現状の直接証拠

### 3.1 強み

- **[事実]** タイトルの夜山、蝕、大燈籠、「八季の命を、継いでゆけ。」は数秒で作品の約束を伝える。
- **[事実]** 世代交代、星契り、誕生、命名、成人、生業、死、辞世、形見、家系図、宿敵、夢、三結末まで実装済み。
- **[事実]** Homeには血脈危機、今月の決断、短中長期の務めがあり、次の行動を推薦できる。
- **[事実]** 帰還後の「人・土地・千年」の三痕、当主継承時の「今代の問い」、形見の初代所有者を保持するデータが既にある。
- **[事実]** 郷、出立、Dungeon/Battleのラスター表現とPC読み幅は改善済みである。

### 3.2 公開版で確認した摩擦

- **[事実]** 新規Homeでは、血脈危機より先に実日付の「日参り」報酬modalが表示された。最初の感情が「一族を救う」より「通貨受取」になる。
- **[事実]** 1280×720のHomeは`scrollHeight=1257`、button 24件。初回画面だけで約1.75画面分の縦長さを持つ。
- **[事実]** 同じ「星契りへ」が危機札、綴の助言、血脈診断、今月の決断に重複する。
- **[事実]** 星契りは180神を初期一覧へ置き、DOM上はbutton 197件、神行180件。契約可能、費用不足、封印中が同じ探索面に混在する。
- **[事実]** 唯一の親を選ばなくても神を先に選べるが、下部は「親を選べ」のままで契約不能となる。
- **[事実]** Homeで下へscrollした状態から星契りへ遷移すると`scrollY=396`が残り、星契りの見出しは`top=-376px`で画面外になった。出立でも同種の再現報告がある。

### 3.3 「物量は深さではない」という証拠

- **[事実]** 装備810点は初期15点＋53系譜×15段で、系譜の効果は`atk / def / bonus`の式展開が中心である。
- **[事実]** 180基礎敵のうち106体は技0、68体は技1、6体は技2。技を持つ敵も通常AIで技を選ぶ確率は35%。若・老は同じ技とAIを継ぐ数値変異である。
- **[事実]** 辞世は性根voiceと3死因からpoolを選び、戦績、事績、親子関係、形見を直接参照しない。公開文言「性根と生き様で変わる」との距離がある。
- **[事実]** 初期の代表ボスは最弱方策でも100%勝利を要求するbalance testがあり、オート戦闘は操作面上部へ常設される。
- **[推定]** 現状の戦闘は、敵の見た目が違っても「オートで流す同じ問題」になりやすい。
- **[事実]** 図鑑は土地・魔性・星神・宿敵を持つが、装備は収集対象に含まれず、家譜の総合収集率も土地・魔性・星神だけで算出される。
- **[事実]** 装備には部位・品質・希少度・入手由来・継承の5軸、稀相由来、初代所有者が既にある一方、一覧は最大810点を個品で扱う。鍛錬は六能力の`現在値 → +3`が中心で、灯型・星脈・家業・次の技・子への継承との関係を一面で説明しない。

### 3.4 計測と正典の欠損

- **[事実]** GoatCounterのsite codeは`your-code` placeholderであり、公開PVもSPA内旅程も計測できない。
- **[事実]** save内には物語sceneの集計があるが、外部送信はなく、初回帰還率や継承率を横断集計できない。
- **[事実]** `regionVisualV2`は現在既定ONだが、GDD内には外部魅力評価と物理端末gate完了までdefault-OFFとする旧契約も残る。
- **[事実]** 年数表示はChronicle/Dungeonで`seasonIndex / 12`、Ending/共有画像で`seasonIndex / 4`となり、記録が3倍ずれる可能性がある。
- **[事実]** README、STATUS、GDD、MARKETINGには神数、敵数、時間単位、公開状態の古い記載が残る。

### 3.5 監査の再現手順

公開版監査は2026-07-21に`https://hitsugi-game.github.io/hitsugi/?audit=20260721`、Chromium、viewport 1280×720、新規saveで実施した。再現順は「はじめから→宿命（標準）→Intro skip→日参りを閉じる→Homeを下へ396px scroll→今月の決断の星契り→先頭神を選択」。各画面でDevTools Consoleから以下を採る。

```js
({ scrollY, innerHeight, scrollHeight: document.documentElement.scrollHeight,
   buttons: document.querySelectorAll('button').length,
   h1Top: document.querySelector('h1')?.getBoundingClientRect().top })
```

Homeは`scrollHeight=1257 / buttons=24`、遷移後の星契りは`scrollY=396 / h1Top=-376`。星契り件数は画面の神一覧row 180件と全button 197件をDOMから数えた。再監査では同じquery付きURL、viewport、手順を使い、screen recording、Console出力、commit SHAを1組の証拠として`docs/qa/`へ保存する。コード由来の件数は`npx tsx`で`ENEMIES`、`ITEM_BASES`をimportして再集計し、敵技率は`src/core/battle.ts`、辞世参照軸は`src/core/epitaph.ts`、年数除数は`rg -n "seasonIndex.*[/%]" src`で照合する。

## 4. 証拠付きの非自明な洞察

### Insight 1: 最大の魅力欠損は美術ではなく「因果の編集」

- **[事実]** 個別機能として星契り、帰還三痕、今代の問い、形見、家譜は存在する。
- **[推定]** これらが別画面・別月・任意帳面へ分散し、プレイヤーには「今の判断が次代へ残った」一本の出来事として届きにくい。
- **[仮説]** 既存データを`約束→行動→結果→継承`の4拍へ再配置する方が、新規イベント100件より魅力改善幅が大きい。
- **最安検証**: 同じsaveから現行版と4拍版の帰還・継承mockを作り、5名へ名前、約束、結果、受け継いだものを再生してもらう。
- **反証条件**: 4/5未満しか因果を再生できない、または現行版との差がない。

### Insight 2: コンテンツ量が初回価値への遮蔽物になっている

- **[事実]** 初回Homeのbutton 24件、星契りの神行180件に対し、初月の本質的行動は「後継を得る」一つである。
- **[推定]** 選択肢の多さが自由ではなく、初見には比較責任と見落とし不安として働く。
- **[仮説]** 初回は「推奨3柱＋すべてを見る」に縮約し、唯一の親を選択済みにすると、自由を失わずに契り完了が速くなる。
- **最安検証**: 現行と縮約版のクリックprototypeを各8名。契り時間、戻る回数、理由説明、自由を奪われた感覚を比較する。
- **反証条件**: 契り時間が30%以上短縮しない、または自由が減ったと答える人が3/8以上。

### Insight 3: 日次報酬は安全でも、作品の第一印象を希釈する

- **[事実]** streak損失はないが、新規Homeで実日付報酬が自動表示され、今日の御題と留守番内職も実時間に結びつく。
- **[推定]** 「短命な家族の約束」より「毎日戻るブラウザゲーム」の文法が先に立つ。
- **[仮説]** 日参りを無報酬の再会文へ、御題をゲーム内月次の務めへ、留守番内職を一定の帰還演出へ置換しても継続は落ちず、自律感と作品固有性が上がる。
- **最安検証**: 実日付版とゲーム内月版の文面・画面A/Bを経験者8名へ提示する。
- **反証条件**: ゲーム内月版支持が6/8未満、または初期資源不足で詰みが増える。

### Insight 4: オートは省略機能ではなく、育成結果を鑑賞する正式な遊び方

- **[事実]** 現行版は戦闘中いつでもオートを入切でき、遠征内で状態を保持し、設定からオート開始を既定にできる。これは短時間ブラウザプレイと810装備・世代育成を支える既存価値である。
- **[推定]** 問題はオート利用率ではなく、オートが通常攻撃中心で「なぜ勝った／負けた」「形見・鍛錬・灯座がどう効いた」が結果から読めないことにある。
- **[仮説]** 全戦闘でオートを常時利用可能にしたまま、`堅実 / 温存 / 全力`の方針、任意の停止条件、戦果の一行理由を加えると、手動を強制せず育成の手応えが上がる。
- **最安検証**: 同一固定戦闘を方針3種で自動実行し、5名が方針差と勝因を説明できるか、途中介入と停止が迷わず行えるかを見る。
- **反証条件**: 4/5未満しか方針差を説明できない、現行より戦闘時間が20%以上伸びる、オート利用者が報酬・発見・進行で不利になる。

### Insight 5: 辞世の量より「その人にしか言えない一行」が価値を持つ

- **[事実]** 1370首の選択は性根と死因が中心で、生涯の具体を参照しない。
- **[推定]** 後半ほど文面の豊富さより、生涯との不一致が目立つ。
- **[仮説]** 辞世本文を大量追加せず、事績、討伐、親子、形見のうち2軸を前置き・後置きの短い「生涯の返歌」へ反映すると、名前の記憶と継承意欲が上がる。
- **最安検証**: 同一人物の現行辞世と具体2軸版を各5例提示し、「この人物だけの言葉」に見える方を盲検比較する。
- **反証条件**: 具体版の選好が4/5未満、またはテンプレート感が強まる。

### Insight 6: 810装備は810枚のカードではなく、53本の系譜として見せる

- **[事実]** 装備は初期15点と53系譜×15段で構成され、品質と希少度は別軸、形見と代重ねで希少度が深まる。
- **[推定]** 個品の長大一覧は所持管理には必要でも、収集達成や「次に欲しいもの」を認識する単位として大きすぎる。
- **[仮説]** 収集面では53系譜を棚札にし、各棚で15段、最高到達、現在所持、形見、由来、次の手掛かりを見せる方が、量を水増しでなく家の歴史として感じられる。
- **最安検証**: 個品gridと系譜棚mockを収集経験者8名へ提示し、所持把握、次目標の発見、形見の想起を比較する。
- **反証条件**: 6/8未満しか次に集めたい一系譜を選べない、検索時間が現行より増える、810点の総数・未所持位置を見失う。

### Insight 7: 鍛錬の魅力は`+3`ではなく「この人をどう育て、何を遺すか」

- **[事実]** 一族には六能力、星脈、灯型、24家業、習得技、装備、事績、子へ継ぐ血潮があるが、鍛錬面の主表示は六つの数値選択である。
- **[推定]** 数値だけでは、今の一手が役割、次の技、装備適性、次代へどうつながるかを判断しにくい。
- **[仮説]** 人物ごとに`今の戦型 / 次の節目 / 次代へ残る血`を一面に置き、理由付き推薦を最大3件だけ示し、六能力の自由選択を残すと、正解の強制なしに育成意図が生まれる。
- **最安検証**: 現行六能力gridと「見立て＋推薦3＋全能力」mockを8名で比較し、選択理由の説明、確定時間、自由を奪われた感覚を測る。
- **反証条件**: 6/8未満しか育成理由を説明できない、推薦集中率が80%以上で他選択が死ぬ、確定時間が現行比30%以上増える。

## 5. 類似事例から外せる制約

### 5.1 Hades: 反復と物語を別レーンにしない

SupergiantのGreg Kasavinは、物語を見たいプレイヤーが難度壁で止まらないよう緩和systemを持たせる設計を説明している。『灯継ぎ』では敗北・帰還・死を、物語の停止ではなく次の関係変化へ使う。ただし単に難しくしないのではなく、手動判断の達成感と物語アクセスを分けて守る。
出典: [Game Developer / Hades narrative design interview](https://www.gamedeveloper.com/design/roguelikes-and-narrative-design-with-i-hades-i-creative-director-greg-kasavin)

### 5.2 Wildermyth: 生成人物への信頼を壊さない

Wildermythの開発者講演は、生成人物への愛着だけでなく、性格逸脱や信頼破壊を避けることを同じ重要度で扱う。『灯継ぎ』でも、性根と無関係な辞世、未回収の親子約束、同じ顔・同じ行動の反復は、追加文章より先に是正する。
出典: [GDC Vault / Getting Players Emotionally Invested in Procedural Characters in Wildermyth](https://www.gdcvault.com/play/1027924/Independent-Games-Summit-Session-Getting)

### 5.3 Rogue Legacy: 初回説明は3〜5分、差異にはtrade-offが必要

Rogue LegacyのTeddy Leeは、独自の死と恒久進行を持つ複雑なゲームでも、導入を最初の3〜5分へ限定したと説明している。またRogue Legacy 2では、traitsを単なる驚きから明確なtrade-offへ変えた。『灯継ぎ』も180神や810装備を最初から説明せず、初回は一つの命脈へ絞り、選択差を数値増加だけでなく代償と物語結果で示す。
出典: [Game Developer / Rogue Legacy tutorial deep dive](https://www.gamedeveloper.com/design/game-design-deep-dive-how-i-rogue-legacy-i-handles-tutorials-without-being-boring)、[Nintendo / Rogue Legacy 2 designer interview](https://www.nintendo.com/us/whatsnew/indie-world-interview-rogue-legacy-2-game-designer-teddy-lee-talks-about-the-newly-available-heir/)

### 5.4 動機づけ: 自律性・有能感・関係性を同時に満たす

ゲーム文脈の研究では、操作習熟は有能感の必要条件であり、難度と能力の不一致は魅力を損なう。ここでの操作習熟は手動戦闘の強制に限らず、育成方針を立て、オート方針を選び、結果を理解して次の編成へ反映することも含む。主経路は、推奨で迷いを減らしても「すべて見る」を残す自律性、育成と結果を結ぶ有能感、一族・神・宿敵が覚えている関係性を同時に満たす。
出典: [Przybylski et al., 2014, Journal of Personality and Social Psychology](https://www.selfdeterminationtheory.org/wp-content/uploads/2015/01/2014_PrzyDeciRigbyRyan_JPSP.pdf)

## 6. 経路選択

| 経路 | 非対称性 | 可逆性 | 情報価値 | 資源適合 | 合計 / 20 |
|---|---:|---:|---:|---:|---:|
| A. 画像・地域・神・敵をさらに増やす | 1 | 2 | 1 | 2 | 6 |
| B. 全画面を均等に磨き続ける | 2 | 2 | 2 | 2 | 8 |
| **C. 初回命脈＋初回継承の縦切りを検証する** | **5** | **5** | **5** | **4** | **19** |
| D. 戦闘を廃し家譜simulationへ即pivot | 3 | 1 | 3 | 2 | 9 |

**主経路はC**。既存コンテンツを削除せず露出を制御し、最も多くの新規プレイヤーが通る一本へ集中する。

戦闘の不可侵条件:

- オートは全ての戦闘で開始前・戦闘中とも利用でき、設定の「最初からオート」を維持する。
- 手動とオートで報酬table、drop率、図鑑発見、称号、血珠、経験・成長処理を分けない。
- 稀相・boss・宿敵で自動停止させない。停止はプレイヤーが任意設定した場合だけ発動し、いつでも再開できる。
- 手動は細かな対象・技・道具選択、オートは方針設計と育成結果の鑑賞という、優劣でなく異なる操作様式として扱う。

代替分岐:

- 初回帰還は高いが継承前に離脱する: 月次反復を圧縮し、日常月をmontage化する。
- オート利用が多い: 制限せず、方針選択、結果の説明、装備・鍛錬との接続を優先する。
- 手動の利用が少ない: 手動専用報酬を置かず、入力数、対象確認、情報密度を改善する。利用率自体を合否にしない。
- 画像blindでAI/寄せ集め指摘が2/8以上: 新規生成を停止し、低密度の統一ink kitまたは専門art ownerへ切り替える。

### 6.1 デザイン方向の比較と採用

| 方向 | 中心比喩 | 強み | 弱み |
|---|---|---|---|
| **A. 煤墨の継承工房** | 得た物・磨いた血を家族の痕として継ぐ | 一族、装備、鍛錬、家譜を一本化できる | skeuomorphismを過剰にすると操作性を損なう |
| B. 六脈の大燈籠 | 六能力を六本の灯芯で育てる | 低コストで育成が分かりやすい | 装備収集と土地の記への接続が弱い |
| C. 夜藪標本庫 | 棚・標本札・採集印で収集を主役にする | collection欲求が強い | 一族より物品が主役になりやすい |

**採用はA**。既存の「墨金の家譜」「拓本帳」「身体＋三装備枠」を壊さず、Bの六脈線を鍛錬へ、Cの標本詳細を図鑑へ限定利用する。自己選択であり、理由は『灯継ぎ』だけの差別化が「強い物を集める」ではなく「誰が何を次代へ残したか」にあるためである。

Brand truth: **強さは数値ではなく、誰かが次代へ残した痕である。** 1 viewport内の大判フルカラー絵は原則1点。人物・神・敵・装備を同時に画集状へ並べず、一覧は低彩度・統一crop、選択中の一点だけ原色と大きな絵へ戻す。新しいAI背景で質感を足さず、罫線、煤、綴じ、朱印、灯の情報層で統一する。

## 7. フェーズ別ロードマップ

### Phase 0: 事実を直す（0〜3日）

作業:

- 画面遷移時のwindow scrollを先頭へ戻す。画面内list scrollだけを画面ごとに保持する。
- `seasonIndex / 4`と`/12`を単一helperへ統一する。
- GDD、STATUS、README、MARKETINGの神数、敵数、時間単位、探索方式、公開状態を現物へ同期する。
- 初見16名を現行版8名と改善mock 8名へ無作為に分けるindependent-groups testを実施し、screen recordingと発話を残す。同じ参加者を両条件へ入れず、操作学習の持越しを防ぐ。
- 外部analytics導入前でも、テストsessionごとに時刻、停止点、戻る回数、介助を手動記録する。

入口: 現HEADをfixture化。
出口: scroll残留0、年数表示一致、文書矛盾0、現行8名の基準線と改善mock 8名の初見結果が揃う。
期限: 72時間。

### Phase 1: 初回命脈を一本化（4〜10日）

作業:

- 新規Homeでは日参りmodalを出さず、危機1件、主CTA1件、他案の折り畳みだけを最初のviewportへ置く。
- 星契りは唯一の親を事前選択。初期面は契約可能な推奨3柱と「全180柱を見る」。
- 推奨理由を「弱い能力を補う」「今代の問いへ沿う」「前代の縁を継ぐ」の文章で示す。
- 初回の編成は唯一の成人を選択済みとし、選び直せる。
- 初遠征の目的を「奉燈を稼ぐ」ではなく、家祖が子へ残す最初の約束として見せる。

入口: Phase 0の離脱点が判明。
出口: 8名中6名が10分以内に星契り→誕生、8名中6名が30分以内に初回帰還または安全中断。自由を奪われた回答2名以下。
分岐: 未達なら美術・中盤・終盤へ進まず、導線だけ再設計する。

### Phase 2: 帰還を一つの物語へする（2週目）

作業:

- 既存`returnTraces`を専用の帰還一面へ集約し、「家族」「土地」「千年」を最大3件だけ表示する。
- 各結果から次の一手を一つ提示し、「ここで休む」を同格で用意する。
- 戦利品詳細は二次導線へ置き、重要装備は前所有者・産地・次に役立つ理由を付ける。
- 郷、Home、家譜の最低2画面へ同じ帰還痕を返す。

入口: 初回帰還率が基準値化済み。
出口: 8名中6名が10秒で変化2件を説明し、60秒以内に次行動か満足終了を選べる。

### Phase 3: 初代の死を作品の核心にする（3〜4週目）

作業:

- 初代だけは確率任せにせず、命名直後の約束、初遠征前後の親子scene、死前の一場面を保証する。
- 辞世に事績、討伐、親子、形見のうち最低2軸を反映する短い「生涯の返歌」を付ける。
- 「今代の問い」を次代の最初の出立、事件、家譜見出しのいずれかで一度だけ回収する。
- 継承後に、先代から変わった能力、形見、未完の約束を一面で見せる。

入口: Phase 2 PASS。
出口: 一世代テスト中央値45分以内、5名中4名が親子名と受継ぎを説明、3名以上が自発的に次月開始。

### Phase 4: 灯史帳と53系譜の収集循環（4〜6週目）

作業:

- 家譜概要を収集hubにし、`新着3件 / あと一歩3件 / 家の逸品3件`から図鑑・形見・装備系譜へ一操作で移る。画面統合はせず既存routeを結ぶ。
- 図鑑は土地、魔性、星神、宿敵を維持し、モバイル詳細を`Sheet`化。閉じると元カードへfocusを返し、前後移動を付ける。
- 魔性の若・常・老を別カードに増やさず、基礎種一項目の「三相印」として既存のraw遭遇IDから発見状況を示す。
- 宝具は810個のflat gridでなく、初期15点と53系譜の棚へ束ねる。棚は最高到達段、所持、装備中、形見、由来、次の手掛かりを持ち、個品検索と全810点表示も二次導線で残す。
- 永続発見記録は`seriesId → 15段bitset`を正とし、表示用の最高到達段はbitsetから導出する。各段の発見bitを個別に保持し、最高段から未発見の下位段を推定しない。由来・形見の具体は現存Itemと家譜へlinkし、旧saveは蔵と全員の装備から安全に初期化する。
- 収集報酬は物語、朱印、額装、称号を先行し、戦闘能力やdrop率へ直結させない。期間限定、streak、claim連打は使わない。

入口: Phase 3 PASS、collection optional fieldのmigration/rollback設計完了。
出口: 8名中6名が10秒以内に新着と次目標を一つずつ説明、6名が53系譜から欲しい棚を選択、モバイルで選択→詳細→閉じる→元focus復帰、390px/200% zoomで横overflow 0。

### Phase 5: 育ちの見立てと全戦闘オート（6〜8週目）

作業:

- 鍛錬上部へ人物一人の`今の戦型 / 次の節目 / 次代へ残る血`を置く。星脈、灯型、家業、習得技、装備、上位2能力から実データだけで導出し、未来を断定しない。
- 理由付きの「今伸ばすなら」を最大3件表示し、六能力2×3と自由選択を常に残す。推薦は`役割を伸ばす / 弱みを補う / 次代へ残す`のどれか一文を必須にする。
- 血珠不足・能力上限でも能力詳細を開ける。CTAだけを理由付きdisabledにし、既存の回数stepper、前後比較、外側click、Escape、focus returnを維持する。
- オートは通常、elite、稀相、主、宿敵、常夜百層の全てで常時ON/OFF可能、設定のオート既定を維持する。解放条件と手動専用報酬は作らない。
- オート方針はまず`堅実（回復優先） / 温存（MP・道具節約） / 全力（弱点・技優先）`の3つ。任意停止は`HP危険 / 新発見 / 稀相 / 主`ごとの設定とし、既定では戦闘を止めない。
- 戦果面へ`方針 / 技・道具の使用 / 形見・鍛錬が効いた一手 / 新発見`を最大4行で返す。新発見modalは出さず、オート進行を中断しない。
- 敵行動型を加える場合も最初は露出上位12体だけ。オートAIと同じ敵意図情報を手動UIにも出し、手動・オートのどちらかを正解扱いしない。

入口: 固定seedで全battle種のauto開始・途中切替・報酬同一formulaを検証。
出口: 全battle種でauto操作到達率100%、手動／autoによる報酬modifier 0、Escまたはtapで次の味方入力前に停止、5名中4名が方針差と勝因を説明、鍛錬8名中6名が選択理由と次代への影響を説明、誤操作0。

### Phase 6: 関係の再会を深める（8〜10週目）

作業:

- 初回露出上位12神だけに、邂逅、親縁、深縁の三幕を実装する。
- 同じ神が前代の名を一度だけ呼ぶ。MAX絵の一括制作は行わない。
- 宿敵は前回の傷、殺めた相手、対策の学習を再戦前後へ返す。
- 地域の石碑・縁起は、次の主の弱点、台詞、経路のどれか一つを変える。

入口: Phase 3/4/5 PASS。
出口: 8名中6名が、神または宿敵が一族を覚えている具体を一つ説明する。

### Phase 7: 拡張かpivotかを決める（10〜12週目）

- 「遊べなくなったらとても残念」の理由が一族・継承へ集中するなら、上位露出の神、敵、地域へ同じcontractを拡張する。
- 物量や日次報酬だけが理由ならPMF未達とし、集客・画像量産を止める。
- 初回継承率40%未満なら、月数、探索階層、Home往復、編成操作を圧縮する。
- オート利用率の高低ではpivotしない。方針差を4/5が説明できない、育成変更が次戦の理解へつながらない場合は、AI複雑化を止めて結果説明と編成見立てを優先する。
- 収集8名中6名が次目標を選べない場合、セットや報酬を増やさず、棚の単位、未知の手掛かり、家譜hubの情報順を再設計する。

## 8. North Starと計測

### North Star

**週次命脈サイクル完了率**
その週に遊んだ匿名セーブのうち、`家譜に残る約束 → 結果 → 次代への継承`を1回以上完了した割合。

起動回数、滞在時間、月送り回数、日次報酬受取はNorth Starにしない。

### 最小イベント

- `new_game_started`
- `mode_selected`
- `intro_exited`（complete / skip、経過秒）
- `home_first_ready`
- `recommended_action_opened`
- `pact_confirmed`
- `birth_scene_resolved`
- `expedition_started`
- `battle_mode_changed`（manual / auto、開始時 / 途中、auto方針。利用率は優劣判定に使わない）
- `first_battle_resolved`（win / flee / loss、開始mode、終了mode、方針、介入有無）
- `expedition_returned`（voluntary / boss / wipe）
- `return_summary_resolved`（continue / stop）
- `collection_entry_discovered`（category、系譜、段。固有名や本文は送らない）
- `collection_entry_opened`（new / goal / search）
- `collection_goal_selected`（category、到達済み割合）
- `training_previewed`（推薦 / 全能力、stat種、回数）
- `training_committed`（推薦一致、stat種、回数）
- `succession_completed`
- `successor_action_completed`
- `lineage_arc_completed`
- `session_ended`（満足終了 / 中断、次意図の有無）

収集しない: 人物名、装備固有名、辞世本文、自由入力、家譜本文、save本文。匿名session ID、相対時間、世代、地域tier、操作種別、系譜ID、能力種に限定し、同意、opt-out、削除方針を先に決める。

GoatCounterを使う場合、単純PVだけでは足りない。SPAの画面遷移と上記の集約eventを明示的に送る。外部送信を採用しない場合も、8名の画面録画と観察記録を必須証拠にする。

## 12. 撤退・転換基準

- 初回導線を2回修正しても初回帰還6/8未満: 全景・中盤・終盤制作を停止し、20〜30分の短編構成へ縮退する。
- 一世代中央値45分超かつ継承後自発継続3/5未満: 寿命月数、必須遠征数、月送りUIを再設計する。
- 全battle種のいずれかでオート不可、報酬modifier、発見除外が生じる: 該当phaseをreleaseせず、現行の常時オートへrollbackする。
- 方針を2回修正しても4/5が差を説明できない: 方針AIの追加を止め、`現行auto＋戦果説明`へ縮退する。オート自体は維持する。
- 系譜棚を2回修正しても6/8が次目標を選べない: collection報酬や項目を増やさず、家譜hubと棚の分類を再設計する。
- 鍛錬推薦で自由喪失3/8以上または単一能力集中80%以上: 推薦を情報提示へ格下げし、六能力自由選択を主面へ戻す。
- 辞世具体化後も「生き様」を4/5が説明できない: マーケ文言を修正し、1370首の追加量産を止める。
- PMF調査40%未満: 広告、日次誘導、終盤量産を行わず、自由記述の最大欠損へ戻る。
- 二周目開始15%未満: NG+をリプレイ性と呼ばず、後日談または一族記録閲覧へ格下げする。

## 13. 最初の72時間

### 0〜24時間

1. `setScreen`と内部直接遷移を通る共通scroll reset契約を作る。
2. 年数helperを一つにし、`/4`と`/12`の期待値をfixtureで確定する。
3. README、STATUS、GDD、MARKETINGの現物差分台帳を作る。
4. 初見16名の募集条件、同意文、30分task、5秒質問を用意し、現行8名／改善mock 8名へ無作為割付する。
5. `normal / elite / rare / boss / nemesis / tower`の全battle種で、auto開始、途中切替、設定既定、報酬・発見共通を守る不変条件test一覧を作る。

### 24〜48時間

1. 現行群8名を観察し、初回30分の基準線を取る。
2. Home「危機1＋主CTA1」、Pact「推奨3＋全件」、唯一親/成人の事前選択mockを完成させる。
3. 日参りあり/なしの第一印象を盲検比較する。
4. 初回帰還までのclick、戻る、介助、時間を記録する。

### 14.3 data/save契約

- Phase 4で`collectionV2?: { itemSeriesBits: Record<string, number>; foundingItemIds: string[] }`を採用する。`?`は旧save読込互換のためだけに残し、runtime migration後は空を含む正規形へ必ず初期化する。各53系譜は15段bitsetで正確な発見を保持し、`最高段だけ`から未発見段を誤推定しない。
- stableな`seriesId`と15個の`baseId`を持つexport manifestを単一情報源にする。旧save移行はinventory＋全人物equipmentをscanし、存在しない過去所持を捏造しない。
- 魔性の三相印は既存`codex.enemies`のraw IDを使い、表示時だけbaseへ畳む。星神、土地、宿敵の既存fieldは変更しない。
- `AutoPolicySettings`は既存`settings.ts`とlocalStorageへversion付きで保存し、save本文へ人物名や戦闘履歴を増やさない。未定義・破損時は現行autoへ安全に戻す。

## 16. 最終判断

『灯継ぎ』に次に必要なのは、さらに豪華な画面でも、さらに多い素材でもない。すでにある死、契り、帰還、形見、家譜を、一人の家祖から次代へつながる短い物語として必ず体験させ、発見した物と磨いた血が次の遠征でどう働いたかを手動でもオートでも理解できる編集である。

その一本が8名の初見プレイヤーへ届くまで、追加物量を魅力改善と呼ばない。届いた後だけ、神、宿敵、地域へ同じ因果contractを拡張する。
